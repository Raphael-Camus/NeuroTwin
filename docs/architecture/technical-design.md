# Technical Design

This document is my implementation-level explanation of the current NeuroTwin baseline.

## Current Capability Contract

Inputs:

- scenario key;
- synthetic ROI-level time series generated by the demo;
- task drive definition;
- candidate intervention vector;
- validation thresholds.

Outputs:

- predicted BOLD-like dynamics;
- functional-connectivity matrix;
- virtual effective-connectivity response;
- objective delta;
- validation ledger;
- next validation packet;
- static dashboard artifacts.

## Why the Baseline Is Simple

I intentionally start with a ridge one-step surrogate. A more complex model would look more impressive, but it would make the workflow harder to inspect. The point of this version is to prove that the project has a clean loop:

```text
generate signal -> fit surrogate -> perturb -> validate -> rank next action
```

Once this loop is stable, I can replace only the surrogate layer without changing the validation and learning contracts.

## Upgrade Path

The next technical upgrades I would make are:

1. Add a BIDS/fMRIPrep ingestion path.
2. Convert ROI extraction and QC into explicit artifacts.
3. Add subject-level splits.
4. Replace heuristic acquisition scoring with Bayesian optimization.
5. Add human review records to the validation ledger.
6. Expose DSVL stages through typed APIs.
